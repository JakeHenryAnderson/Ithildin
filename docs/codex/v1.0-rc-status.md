## Implemented Local-Preview Surface

- MCP ingress through the governed path.
- Narrow filesystem read/search/stat/list tools.
- Stored patch proposal and approval-gated patch apply.
- Exact-allowlist GET-only HTTP fetch.
- Bounded read-only Git metadata tools.
- Bounded read-only project metadata summaries, including `project.risk.summary`.
- Bounded approval-gated sandbox artifact text writes through `sandbox.artifact.write_text`.
- Local Agent Run/evidence surfaces and review-console operator views.
- Local audit, verification, and locally signed export evidence.
- Release, review, packet, redaction, and no-new-powers gates.

## Still Blocked Or Deferred

- shell execution;
- Docker socket access;
- Kubernetes tools;
- browser automation;
- arbitrary HTTP methods, headers, bodies, cookies, or broad network access;
- broad filesystem writes, deletes, moves, chmod, archive extraction, or unbounded traversal;
- production identity, enterprise RBAC, hosted MCP, remote MCP, runtime Postgres, hosted telemetry,
  or managed model serving;
- sandbox orchestration, SIEM adapters, compliance automation, custody-grade audit, or public
  security-product positioning;
- plugin SDK or marketplace behavior.

## v1.0 RC Exit Criteria

The project can be treated as a v1.0 local-preview RC candidate only when:

- `make release-check` passes from a clean tree;
- `make review-candidate` passes and packet redaction reports `findings: 0`;
- `make v1-rc-status-check` and `make v1-rc-roadmap-check` pass;
- tool count remains `24` unless a later explicit implementation decision updates the entire
  capability/review surface;
- no critical/high findings remain open for the local-preview boundary;
- public/security-product positioning remains blocked;
- install, demo, workbench, evidence, and shutdown instructions are coherent without relying on
  historical packet archaeology.
