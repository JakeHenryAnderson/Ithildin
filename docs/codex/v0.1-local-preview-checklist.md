# v0.1 Local Preview Checklist

Use this checklist before tagging or handing off a local-preview build.

## Required Checks

- Run `make release-check`.
- Run `make release-evidence` and attach the output, or attach
  `docs/codex/v0.1-release-evidence.md` plus a fresh final evidence output.
- Run `make release-packet` and attach the output for v0.2 external/code review handoff.
- Read `docs/codex/threat-model-and-non-goals.md`.
- Read `docs/codex/v0.2-review-response-and-rc-cleanup.md` when requesting a v0.2 review.
- Read `docs/codex/v0.2-review-packet.md` when requesting a v0.2 review.
- Read `docs/codex/v0.2-external-review-prompt.md` before sending the packet to a reviewer.
- Read `docs/codex/v0.1-public-preview-release-notes.md`.
- Review `docs/codex/mcp-client-examples.md` before publishing MCP client snippets.
- Review `docs/codex/mcp-inspector-recipes.md` before demoing MCP Inspector flows.
- Review `docs/codex/v0.1-security-test-matrix.md` before tagging.
- Run `make manifest-lock-check`.
- If using OPA, verify `/policy/status` reports a verified bundle.
- Run `make demo-seed`, `make compose-up`, `make compose-smoke`, and `make demo-flow`.
- Open the review console and confirm tools, policy preview, approvals, audit integrity, and export work.
- Run `make ollama-smoke`; a skip message is acceptable on machines without Ollama or local models.
- Confirm `/system/status` reports SQLite runtime storage, manifest lock status, principal registry status, audit head, and telemetry status.
- Confirm `/system/status` reports any dev-token or dangerous-config warnings.
- Confirm public-preview warning labels remain visible in README, release notes, generated docs, and
  the review console when relevant.

## Safety Review

- Confirm `.env`, `var/`, `workspaces/demo/`, `site/`, and UI build output are ignored.
- Confirm Docker Compose does not mount `/var/run/docker.sock`.
- Confirm no shell, Docker, Kubernetes, broad filesystem write, or arbitrary network tools were added.
- Confirm `http.fetch` remains exact-allowlist GET-only.
- Confirm the security matrix covers HTTP SSRF, redirect, DNS, proxy, timeout, and size-limit cases.
- Confirm filesystem and patch paths still reject symlinks, `.git` internals, hidden/sensitive paths,
  binary targets, unsupported encodings, and stale bases.
- Confirm the security matrix covers filesystem and patch traversal, link, encoding, stale-base,
  and atomic-write cases.
- Confirm Postgres is readiness-only and not advertised as a runtime backend.
- Confirm OpenTelemetry is opt-in and does not export prompts, arguments, file contents, diffs, responses, or secrets.

## Deferred Beyond v0.1

- Production identity integrations: OIDC, SAML, SCIM, hosted sessions, and multi-tenant stores.
- Runtime Postgres storage adapters and migrations.
- Hosted telemetry collectors and production observability dashboards.
- Kubernetes deployment and agent-facing Docker/Kubernetes tools.
- External notarization, hosted supply-chain signing, and production key-management policy.
- Managed model serving or LLM proxy workflows.
