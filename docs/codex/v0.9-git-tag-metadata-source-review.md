# v0.9 git.show.tag_metadata Source Review Handoff

Status: source-review handoff for one bounded read-only Git tag metadata capability.

This document records the source-review package for `git.show.tag_metadata`. The tool is local-only,
read-only, fixed-argv, and limited to safe tag metadata counts and response-local tag handles. It
does not approve broader capability expansion or a new powerful tool class.

## Handoff Command

```bash
make git-tag-metadata-source-review-bundle
```

Default ignored output:

```text
var/review-packets/v0.9/git-tag-metadata-source-review/
```

## Review Scope

The bundle asks reviewers to inspect:

- the strict manifest/schema shape for `selector.kind: all_local_tags`, bounded `limit`, and
  `additionalProperties: false`;
- the fixed Git `for-each-ref` implementation over local `refs/tags` only;
- workspace toplevel confinement for preview and runtime;
- response safety: no raw tag names, no stable tag-name hashes, no tag messages, no signatures, no
  file contents, no raw diffs, no remotes, and no raw stderr;
- safe handling of lightweight tags, annotated tags, non-commit targets, malformed selectors,
  ambiguous/casefold-conflicting names, oversized output, and parent-repository escape;
- policy preview/runtime parity for the normalized `git_tags` resource;
- MCP exposure through the governed pipeline;
- audit metadata limited to selector/count/truncation/skipped/output-policy evidence.

## Non-Goals

This handoff does not approve raw tag names, tag messages, signatures, stable tag-name hashes,
arbitrary Git commands, caller-controlled format strings, checkout, branch/tag mutation, remote
fetch, file contents, raw diffs, shell execution, broad filesystem access, public/security-product
positioning, or future governed tool powers.
