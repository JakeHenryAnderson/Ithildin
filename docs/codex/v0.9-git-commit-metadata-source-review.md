# v0.9 git.show.commit_metadata Source Review Handoff

This document records the source-review handoff and internal xhigh review result for the bounded
v0.9 `git.show.commit_metadata` implementation. It prepares the new read-only Git metadata tool for
optional external/source review. It does not approve future capability expansion or
public/security-product positioning.

## Review Boundary

- Area: `git-commit-metadata`.
- Finding namespace: `EXT-GITMETA-###`.
- Source-review status: internally xhigh-reviewed and locally acceptable for continued
  local-preview development; external/source disposition optional/deferred.
- Runtime boundary: v0.1 local-preview.
- Capability boundary: exactly one bounded read-only Git metadata tool.

The handoff bundle is generated with:

```sh
make git-commit-metadata-source-review-bundle
```

The ignored output directory is:

```text
var/review-packets/v0.9/git-commit-metadata-source-review/
```

## Review Questions

Internal xhigh review has already decided that `git.show.commit_metadata` can continue for the
local-preview runtime boundary. Optional external review, if requested later, should focus on:

- structured ref validation and denial of ambiguous Git revision syntax;
- fixed Git argv behavior and no shell/checkout/remote-fetch/mutation behavior;
- bounded metadata output, email redaction, untrusted metadata labeling, and safe decoding;
- hidden/sensitive changed-file path redaction;
- no raw diffs, file contents, blob contents, or broad filesystem exposure;
- policy preview/runtime parity for the syntactic `git_commit` resource;
- MCP exposure and role visibility through the existing governed pipeline;
- no-new-powers and tool-surface gates staying limited to this one read-only addition.

## Internal xhigh Review Result

Internal xhigh source review found no critical findings and seven actionable findings across two
passes:

- `SUB-GITMETA-001` high: commit identity parsing/privacy. Fixed by replacing in-band separator
  parsing with fixed per-field Git metadata reads, timestamp validation, identity-name
  sanitization, and email-like name redaction.
- `SUB-GITMETA-002` high: parent-repository workspace scope. Fixed by failing closed unless the Git
  toplevel is within or equal to the configured workspace root, for both runtime execution and
  resource construction.
- `SUB-GITMETA-003` medium: sensitive changed-path redaction. Fixed by expanding credential and
  private-key path classification.
- `XH-GITMETA-001` medium: source-review packet hash traceability. Fixed by hashing the exact
  normalized implementation-packet artifact and asserting hash consistency in release-readiness
  tests.
- `XH-GITMETA-002` medium: Git subprocess output bounding and identity-name limits. Fixed by
  reading stdout incrementally to the configured bound, killing/truncating safely, suppressing
  stderr, and capping sanitized identity names.
- `XH-GITMETA-003` low: expanded negative ref validation coverage. Fixed with explicit tests for
  revision operators, pathspec-like refs, remote refs, control characters, non-NFC refs, and
  non-commit object IDs.
- `XH-GITMETA-004` low: stable hash privacy caveat. Fixed by documenting stable hashes as local
  evidence aids rather than anonymity guarantees.

With those findings fixed and focused/full gates passing, internal xhigh review is sufficient to
continue local-preview development for this bounded read-only lane. This is not public/security-
product approval and does not approve broader capability implementation.

## Closure Boundary

Even if this source-review lane closes, it does not approve:

- arbitrary Git execution;
- caller-supplied Git argv, format strings, pathspecs, checkout, remotes, or credentials;
- raw diffs, file contents, blob contents, symlink targets, or broad filesystem access;
- write behavior, approval changes, production identity, hosted telemetry, runtime Postgres, remote
  MCP, public/security-product positioning, or future governed tool powers.

`git.show.commit_metadata` remains local-preview read-only metadata mediation, not a Git sandbox.
