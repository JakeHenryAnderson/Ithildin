# v0.4 External Review Prompt

Use this prompt with GPT 5.5 Pro / Very High or a human expert after running `make review-candidate`
from a clean tree.

## Context

Ithildin v0.4 is a review-closure and evidence-maturity wave over the same v0.1 local-preview
runtime boundary. It did not add shell, Docker socket, Kubernetes, browser automation, arbitrary
HTTP, broad filesystem writes, production identity, runtime Postgres, hosted telemetry, remote MCP,
or a plugin SDK.

## Please Review

Review the attached packet as a review/evidence handoff first, and as source-review guidance second
unless source files are included. Please distinguish:

- packet/evidence confidence;
- implementation/source confidence;
- documentation/positioning risk;
- blockers to any future capability expansion.

## Focus Areas

1. Patch apply state machine, approval binding, recovery evidence, and diagnostics.
2. Filesystem path/race contract, macOS/Linux support claims, and Windows/WSL deferral.
3. HTTP fetch canonicalization, DNS/IP/redirect/proxy behavior, and safe-error boundaries.
4. Signed audit export and signed manifest-lock verification as local evidence only.
5. Policy preview/runtime parity and YAML-vs-OPA boundary.
6. MCP ingress thinness and fixed local principal/session assumptions.
7. Review console approval evidence and failure/unauthorized states.
8. Redaction evidence boundary, packet redaction scan, and leak-claim wording.
9. Review-run/finding/closure machinery and whether the closure matrix is honest.

## Decision Questions

- Is the v0.4 packet ready for an external/source review handoff?
- Are any critical/high findings visible from the packet alone?
- Which areas must receive adversarial source review before new tool powers?
- Does the capability-decision seed correctly defer dangerous powers?
- What would you require before considering v0.5 capability expansion?

## Required Warning

Do not treat v0.4 as production security software, OS sandboxing, enterprise identity, immutable
audit, hosted trust, or remote MCP readiness.
