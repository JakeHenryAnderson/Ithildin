# v0.5 Roadmap From v0.4 Review

This document preserves GPT 5.5 Pro's v0.4 roadmap/gating review as actionable planning input for
Tasks 152-180. It is not a source-code audit and does not approve capability expansion.

## Review Judgment

The v0.4 direction should continue. The important instruction is to keep assurance before
capability: make blockers explicit, keep packet diffs visible, preserve no-new-power boundaries, and
require a separate capability-expansion decision after external/source review.

The review endorsed the existing boundary:

- v0.4 is review closure, evidence maturity, diagnostics, and local-preview hardening;
- the runtime boundary remains v0.1 local preview;
- internal AI/subagent review can pressure-test but cannot close external review rows;
- external GPT 5.5 Pro / Very High or human expert review remains required before any new powerful
  tool class or production/security-product positioning.

## Directives Carried Forward

1. Keep every task checkpoint committed separately.
2. Run focused tests for touched surfaces and `make release-check` before each checkpoint commit.
3. Run `make review-candidate` and packet diff gates after each wave.
4. Preserve the tool list and deferred-power list unless a user-approved boundary decision says
   otherwise.
5. Add explicit blockers for future capability expansion, broader distribution, and external
   handoff.
6. Treat Task 151 as a review packet only; it does not unlock v0.5 powers.

## v0.5 Purpose

Tasks 152-180 are a source-review and capability-decision preparation wave. They should turn the
v0.4 packet into stronger review closure machinery and explicit go/no-go evidence. They should not
add shell, Docker socket, Kubernetes, browser automation, arbitrary HTTP, broad filesystem writes,
production identity, runtime Postgres, hosted telemetry, remote MCP, plugin SDKs, or new governed
tool powers.

## Wave Gates

After each wave, run:

```sh
make release-check
make review-candidate
make reviewer-findings-check
make review-run-manifest-check
make review-packet-diff-gate OLD=<prior checkpoint> NEW=<current checkpoint>
git status --short
```

If any critical/high finding is open, if the tool list changes, if deferred powers change, or if a
task requires changing the local-preview boundary, stop and record a boundary decision before
continuing.
