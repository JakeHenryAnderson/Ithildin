# v0.7 Design-Only Capability Rubric

Task 208 creates a design-only rubric for later capability discussion.

A candidate capability may be discussed only if it preserves the current deferred-power list during
v0.6 and is labeled design-only.

## Required For Any Future Implementation

- External/source review closure: complete for relevant current lanes.
- Capability expansion: explicitly changed from no-go by a later decision.
- Public/security-product positioning: separately reviewed.
- No new governed tool powers in v0.6.

Disallowed for v0.6: shell, Docker socket, Kubernetes, browser automation, arbitrary HTTP,
broad filesystem writes, plugin SDKs, production identity, runtime Postgres, hosted telemetry,
remote MCP, or hosted trust roots.
