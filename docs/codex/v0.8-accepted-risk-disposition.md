# v0.8 Accepted-Risk Disposition

Wave 2 converts the accepted-risk register from a mostly pending inventory into an explicit
v0.8 product-risk decision artifact.

## Decision Summary

| Result | IDs | Meaning |
| --- | --- | --- |
| `closed_local_preview` | AR-004, AR-005, AR-006, AR-007, AR-008 | Lane review closes the risk only for the v0.1 local-preview behavior named in the record. |
| `accepted_deferred` | AR-001, AR-002, AR-003, AR-009, AR-010 | The risk is intentionally carried forward and blocks stronger claims or capability classes until revisited. |

No accepted risk approves production identity, production security positioning, runtime Postgres,
hosted telemetry, remote MCP, shell/Docker/Kubernetes/browser tools, arbitrary HTTP, broad
filesystem writes, plugin SDKs, hosted trust roots, custody-grade evidence, or external
notarization.

## Closed Local-Preview Risks

- `AR-004` is closed for stored, approval-gated patch apply only.
- `AR-005` is closed for macOS/Linux local-preview filesystem/platform claims only.
- `AR-006` is closed for exact-allowlist GET-only HTTP fetch behavior only.
- `AR-007` is closed for YAML-canonical local-preview policy/registry behavior only.
- `AR-008` is closed for local signed-evidence and verification behavior only.

## Accepted-Deferred Risks

- `AR-001` keeps sandbox, EDR/MDM, production-security, and host-compromise-protection claims
  blocked.
- `AR-002` keeps production identity, enterprise RBAC, multi-user authorization, OIDC, SAML, SCIM,
  and remote MCP identity claims blocked.
- `AR-003` keeps custody-grade evidence, external notarization, compliance audit, and stronger
  audit-integrity claims blocked.
- `AR-009` keeps runtime Postgres, multi-user storage, hosted retention, and production storage
  claims blocked.
- `AR-010` keeps guaranteed redaction, secrets-manager support, and secret-safe telemetry claims
  blocked.

## v0.8 Implications

- Limited technical-preview sharing is `conditional_go` under the separate v0.8 public-preview
  claims decision and remains warning-packet/local-preview scoped.
- Public/security-product positioning remains blocked/no-go.
- Capability implementation remains blocked.
- Capability design may consider only documented proposals that account for accepted-deferred risks.
- Any future capability touching secrets, identity, remote deployment, broad network access, or broad
  writes must return to external review before implementation.
