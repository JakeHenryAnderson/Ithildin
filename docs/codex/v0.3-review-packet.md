# Ithildin v0.3-Prep Review Packet

This is the v0.3-prep external/source-review packet for the v0.1 local-preview runtime boundary.
It supersedes the earlier v0.2 packet for the next handoff, while preserving the same narrow runtime
claim: local-first governed MCP/tool mediation, not OS sandboxing or production security software.

## What Changed Since v0.2

The v0.3-prep track did not add governed tool powers. It added implementation-assurance,
fail-closed validation, review automation, and handoff evidence:

- source-review closure matrix v2 and reviewer finding intake automation;
- patch-apply crash/failure simulation and state-machine documentation;
- filesystem adversarial race harness and contract enforcement;
- HTTP canonicalization tests, HTTP executor contract, and SUB-001 remediation record;
- signed-evidence replay/substitution tests and evidence-contract versioning;
- policy preview/runtime parity harness and OPA parity decision;
- MCP ingress bypass audit;
- review-console assurance polish for approval evidence, trust state, and diagnostics;
- expanded negative transcripts;
- release evidence schema validation and review packet diff command;
- executor contract set;
- manifest, principal, workspace, and audit adversarial/fail-closed suites;
- expanded release guardrails;
- internal AI review packet v2.

## Required Local Gates

Run these from the repo root before external handoff:

```sh
make review-candidate
make release-check
make internal-review-packet
git status --short
```

`make review-candidate` regenerates release evidence, signed-evidence demo fixtures, negative
transcripts, review bundle, consolidated packet, and docs site. `git status --short` should be
empty after committed checkpoint work.

## Reviewer Focus

Ask the reviewer to inspect source and tests for:

- patch apply approval binding, recovery evidence, and no-partial-write behavior;
- filesystem path/open/race semantics on macOS/Linux only;
- HTTP fetch allowlist, redirect, DNS/IP, pinned transport, proxy suppression, and safe errors;
- audit chain verification, signed export verification, and evidence replay/substitution behavior;
- manifest lock validation and optional signed-lock enforcement;
- principal/workspace registry fail-closed behavior;
- policy preview/runtime parity and OPA/YAML boundary clarity;
- MCP ingress thinness and fixed-principal behavior;
- review-console evidence visibility and failure-state clarity;
- release evidence schema, guardrails, bundle hashes, and packet diff outputs.

## What This Packet Does Not Prove

This packet does not prove production readiness, OS isolation, external audit custody, immutable
storage, hosted supply-chain signing, production identity, runtime Postgres support, remote MCP
security, or arbitrary tool safety. It is evidence for a local-preview source-review candidate.

## Do Not Add Yet

Do not add shell execution, Docker socket access, Kubernetes tools, browser automation, arbitrary
HTTP methods/headers/bodies, broad network access, broad filesystem writes/deletes/moves/chmod,
archive extraction, secrets-manager tools, plugin SDK/marketplace, remote hosted MCP, production
identity integrations, runtime Postgres, hosted telemetry collectors, or managed model serving until
external/source review closes the current boundary.
