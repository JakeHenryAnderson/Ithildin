# v0.6 Patch Apply External Review Execution

This note starts the first v0.6 external/source-review execution lane. It prepares a source-level
handoff for `fs.patch.apply`, Ithildin's only local-preview write path. It does not close external
review, create findings, approve public preview, or approve capability expansion.

## Command

```bash
make v06-patch-apply-review-packet
```

Direct form:

```bash
uv run python scripts/patch_apply_external_review_packet.py
```

The generated ignored packet is written to:

```text
var/review-packets/v0.6/patch-apply-external-review/
```

The command fails if the working tree is dirty. It also regenerates the v0.6 dispatch packet set so
the patch-apply packet, commit, whole-file hash, and payload hash are current.

The generated patch-apply lane packet supersedes the earlier generic v0.6 dispatch entry for patch
apply. Review findings for this lane should reference the generated packet's commit and dispatch
hash, not an older generic dispatch packet.

## Packet Contents

- `00_PATCH_APPLY_EXTERNAL_REVIEW_INDEX.md` - packet order, commit, dispatch hashes, and boundary.
- `01_PATCH_APPLY_EXTERNAL_REVIEW_PROMPT.md` - ready-to-send source-review prompt.
- `02_PATCH_APPLY_DISPATCH_PACKET.md` - generated focused dispatch packet for patch apply.
- `03_PATCH_APPLY_SOURCE_BUNDLE.md` - primary implementation source files.
- `04_PATCH_APPLY_TESTS_BUNDLE.md` - focused tests for proposals, approvals, governed calls, and
  security regressions.
- `05_PATCH_APPLY_CONTRACTS_BUNDLE.md` - checklist, state machine, filesystem contract, and handoff
  context.
- `06_PATCH_APPLY_INTAKE_COMMANDS.md` - raw-response path and normalization command.
- `patch-apply-review-artifact-hashes.json` - SHA-256 and byte count for the packet files.

The artifact-hash file intentionally hashes the seven markdown handoff files and excludes itself to
avoid a self-referential digest.

## External Response Intake

Store the raw response at:

```text
var/review-runs/v0.6/patch-apply/raw-response.md
```

Then normalize it with the command printed in `06_PATCH_APPLY_INTAKE_COMMANDS.md`.

The patch-apply lane uses finding IDs of the form:

```text
EXT-PA-###
```

The normalizer and finding validators accept this lane-specific namespace. Normalization does not
write finding records, mutate the closure matrix, close external review, or approve new powers.

After normalization and any structured finding updates, run:

```bash
make reviewer-findings-check
make review-findings-summary
make external-review-closure-gate
make release-check
```

## Required Review Access

Patch apply is implementation-sensitive. A review can support source-row closure only when the
reviewer performs `source-level` or `packet-and-source` review and cites affected files/functions.
Packet-only review may still produce documentation or process findings, but it cannot close the
patch apply implementation row.

## Stop Conditions

Stop unrelated work if the patch-apply reviewer reports any critical/high finding. Convert the
finding into a structured `EXT-PA-###` record, fix or disposition it, run focused tests plus
`make release-check`, and update the closure matrix only after external evidence exists.

## Non-Goals

This task does not add broad writes, delete/move/chmod/archive operations, repair endpoints, shell
execution, Docker/Kubernetes/browser tools, production identity, runtime Postgres, hosted telemetry,
remote MCP, plugin SDKs, public-preview approval, or capability expansion.
