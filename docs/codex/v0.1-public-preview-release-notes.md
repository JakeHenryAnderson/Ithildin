# v0.1 Public Preview Release Notes

Ithildin v0.1 is a local-preview governed MCP/tool gateway for security-conscious developers. It
lets an MCP-capable agent use narrow filesystem, git, patch, and allowlisted HTTP tools through
schema validation, deny-default policy, approval, redaction, and tamper-evident local audit.

Ithildin v0.1 is a local-preview mediation layer for AI-agent tool use. It is not a sandbox,
EDR/MDM agent, SIEM, production identity system, hosted MCP platform, compliance audit system, or
immutable evidence store. It assumes the local host, local admin, trusted tool manifests, and local
policy files are part of the trusted computing base. Audit records are tamper-evident local
evidence, not notarized or custody-grade logs.

v0.1 supports narrow built-in tools only: workspace reads, git reads, stored patch proposals,
approval-gated patch apply, and exact-allowlist GET-only HTTP fetch. It deliberately does not
support shell execution, Docker socket access, Kubernetes tools, browser automation, arbitrary HTTP
methods/headers/bodies, broad filesystem writes, plugin marketplaces, hosted MCP, production
identity, or hosted telemetry.

## Warning Labels

- Not production security software.
- Not a kernel sandbox, EDR, MDM, SIEM, production identity system, hosted MCP gateway, or model
  serving layer.
- Local principals are attribution labels and local policy inputs, not enterprise authentication.
- The audit log is local and tamper-evident, not externally notarized or tamper-proof.
- Redaction is best-effort leak reduction, not a guarantee that secrets cannot be exposed.
- `http.fetch` is GET-only and exact-allowlist-only.
- SQLite is the only v0.1 runtime storage backend. Postgres is readiness/status evidence only.
- OpenTelemetry is opt-in preview metadata and must not export prompts, arguments, file contents,
  diffs, responses, or secrets.

## What Is Included

- Stdio MCP adapter backed by the same governed pipeline as the API.
- Hash-pinned trusted YAML tool manifests.
- Local principal registry and role-aware tool visibility.
- Deny-default YAML policy with optional verified OPA bundle evidence.
- Approval-gated stored patch application with immutable action-evidence binding.
- Workspace-scoped read tools and stored-proposal-only patch writes.
- Exact-allowlist, GET-only `http.fetch` with redirect and DNS/IP safety checks.
- SQLite audit storage plus hash-chained JSONL verification and export.
- Local review console for system trust status, policy preview, approvals, patches, and audit.
- Docker Compose local demo stack without Docker socket mounts or privileged containers.
- Static local docs site generation.

## Public Preview Gate

Before tagging or publishing a public preview:

```sh
make release-check
make release-evidence
make release-packet
make review-packet-bundle
```

Also read [Threat Model and Non-Goals](threat-model-and-non-goals.md), confirm `.env` uses a unique
admin token generated with `make admin-token-generate`, review the
[v0.1 Security Test Matrix](v0.1-security-test-matrix.md), and verify the review console shows no
unexpected dangerous-configuration warnings. For v0.2 external/code review, start with
[v0.2 Review Packet](v0.2-review-packet.md).

## Deliberately Deferred

- Production authentication: OIDC, SAML, SCIM, sessions, and multi-tenant identity stores.
- Runtime Postgres storage and production migrations.
- Hosted telemetry collectors and production observability dashboards.
- Remote MCP hosting, Streamable HTTP MCP, and hosted control-plane workflows.
- Shell execution, Docker socket access, Kubernetes tools, browser automation, arbitrary HTTP
  methods/headers/bodies, secrets-manager tools, plugin SDKs, and marketplaces.
- Cryptographic signatures, notarized audit exports, and immutable storage.
