# Ithildin v1.0 RC External Review Prompt

Status: v1.0 local-preview external handoff prompt.

Use this prompt with GPT 5.5 Pro / Very High or a human expert reviewer after generating the v1.0
RC packet with `make review-candidate`. This prompt does not add runtime behavior, tool manifests,
executors, policy rules, API/MCP behavior, UI runtime behavior, sandbox orchestration, SIEM
adapters, production identity, runtime Postgres, hosted telemetry, remote MCP, plugin SDK behavior,
compliance automation, or new governed tool powers.

The intended review mode is v1.0 local-preview release-candidate packet review. If source files are
not attached, the reviewer should distinguish packet/evidence confidence from implementation
confidence.

Reviewers must distinguish packet/evidence confidence from implementation confidence.

## Prompt

```text
You are reviewing Ithildin as an expert external reviewer. Treat this as a v1.0 local-preview
release-candidate packet review unless source files are also supplied.

Context:
- Ithildin is a local-first governed MCP/tool gateway and operator workbench for mediated AI-agent
  tool use.
- The runtime boundary remains local preview. Ithildin is not production identity, not a hosted MCP
  platform, not a kernel sandbox, not EDR/MDM/SIEM, not compliance automation, and not custody-grade
  audit.
- The governed tool count is 24.
- The latest implemented governed tool is sandbox.artifact.write_text.
- No next capability is selected.
- Capability expansion is feature-frozen for v1.0 RC.
- Public/security-product positioning remains blocked.
- Production/security/compliance positioning remains blocked.
- Internal AI/subagent review is supporting evidence only; do not treat it as independent external
  validation.

Please review:
1. Whether the v1.0 RC packet is coherent enough for local-preview release-candidate handoff.
2. Whether install, demo, workbench, evidence, assurance, and shutdown instructions are clear enough
   for a technical reviewer or trusted local-preview user.
3. Whether any wording overclaims production security, sandboxing, custody-grade audit, compliance
   automation, SIEM behavior, hosted trust, production identity, or public/security-product
   readiness.
4. Whether the feature-freeze boundary is clear: no new manifests, executors, policy powers,
   MCP/API behavior, UI runtime controls, sandbox orchestration, SIEM adapters, or product
   positioning expansion without a later explicit post-RC decision.
5. Whether external/source-review pending rows, accepted-deferred risks, and non-goals remain
   visible instead of being described as closed.
6. Whether generated evidence, packet hashes, command transcripts, demo artifacts, and redaction
   evidence are sufficient for a packet/evidence review.
7. Whether any blocker should stop v1.0 local-preview RC labeling.
8. What should remain deferred beyond v1.0.

Please answer in this structure:
- Overall judgment.
- Blockers before v1.0 local-preview RC labeling.
- Should-fix items before handoff.
- Documentation/wording risks.
- Evidence or packet gaps.
- Runtime/security concerns that require source review.
- What is acceptable for local-preview RC.
- What remains explicitly deferred.
- Recommended next post-RC roadmap.
- Do-not-add-yet list.
- Brutal short version.

Important boundaries:
- Do not approve production/security/compliance positioning.
- Do not approve public/security-product positioning.
- Do not approve shell execution, Docker socket access, Kubernetes tools, browser automation,
  arbitrary HTTP, broad filesystem writes, sandbox orchestration, SIEM adapters, production
  identity, runtime Postgres, hosted telemetry, remote MCP hosting, plugin SDK behavior, secrets
  manager tools, or managed model serving.
- If you cannot inspect source, distinguish packet/evidence confidence from implementation
  confidence.
```

## Non-Goals

This prompt is not approval to add runtime powers or make production/security claims. It exists to
ask an external reviewer whether the current local-preview RC packet is honest, coherent, and
sufficiently bounded.
