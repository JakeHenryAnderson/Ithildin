# Ithildin v1.0 Observed Operator Trial Evidence

Status: optional observed local-preview operator trial evidence for the v1.0 RC path.

This document describes a checked evidence layer for an operator-run local demo. It does not add
runtime behavior, tool manifests, executors, policy rules, API/MCP behavior, UI runtime behavior,
sandbox orchestration, SIEM adapters, production identity, runtime Postgres, hosted telemetry,
remote MCP, plugin SDK behavior, compliance automation, or new governed tool powers.

## Purpose

The operator trial record captures readiness without starting services. The observed operator trial
evidence captures whether an intentional `make demo-flow` run has produced a secret-free
`DEMO_FLOW_RESULT.md` artifact and whether that artifact proves the mediated demo completed.

Generate or validate the observed evidence with:

```sh
make v1-operator-trial-observed
make v1-operator-trial-observed-check
```

The generated output is ignored local evidence under:

```text
var/review-packets/v1.0/operator-trial-observed/
```

## Observed Evidence

When `DEMO_FLOW_RESULT.md` exists, the check validates that it records:

- `scenario: guided_local_demo`;
- `demo_step: mediated_patch_flow`;
- an approval ID and proposal ID;
- `patch_apply_status: completed`;
- at least one candidate Agent Run ID;
- `audit_verification_valid: true`;
- audit head hashes for verification and export evidence;
- no file contents, unified diffs, response bodies, secrets, or raw sensitive paths.

When `DEMO_FLOW_RESULT.md` is absent, the check remains valid with status `not_run` because the live
demo is optional and intentionally mutates only the ignored demo workspace.

## Boundary

This evidence only covers Ithildin-mediated demo activity. It does not prove OS isolation, host
compromise resistance, VM/container security, SIEM custody, compliance automation, production
security, external notarization, or activity outside Ithildin-mediated actions.
