# v0.8 Final Decision Packet

Wave 6 records the final v0.8 product-risk decisions after focused implementation lanes closed for
the v0.1 local-preview runtime boundary, accepted risks were dispositioned, public-preview claims
were reviewed, capability design was separated from implementation, and `SUB-080` received a focused
review-console interaction-test harness.

This packet is a decision artifact only. It does not add governed tool powers, approve public or
security-product positioning, implement production identity, enable runtime Postgres, add hosted
telemetry, expose remote MCP, or broaden filesystem/network execution.

## Required Warning Summary

Any shared v0.8 packet, README, quickstart, release note, or UI status text must preserve these
warnings:

- local preview only;
- not a sandbox;
- not production security;
- not production identity;
- not immutable or custody-grade evidence;
- not external notarization;
- not hosted MCP;
- redaction is best-effort leak reduction;
- macOS/Linux local-preview filesystem support only;
- no shell, Docker socket, Kubernetes, browser automation, arbitrary HTTP, broad filesystem writes,
  plugin SDK, hosted telemetry, runtime Postgres, or new governed tool powers.

## Capability Planning Rule

Design-only capability planning may begin only as written proposals. A proposal must include:

- executor contract;
- policy fixtures and policy-preview expectations;
- audit fields and redaction behavior;
- UI/review evidence when admin or approval flows are involved;
- negative transcripts;
- resource limits;
- accepted-risk impact analysis;
- no-new-powers boundary analysis;
- external/source review requirement before implementation.

Capability implementation remains blocked. Design work must not add tool manifests, executors,
policy rules, MCP exposure, approval behavior, production identity, storage backends, telemetry
exporters, network transports, or broad filesystem access.

## Sharing Boundary

Limited technical-preview sharing is allowed only with the warning packet and local-preview framing.
Broad public/security-product positioning remains blocked. Ithildin must not be presented as:

- production security software;
- a sandbox or endpoint protection product;
- not enterprise identity or RBAC;
- not compliance or custody-grade audit;
- not immutable or notarized evidence;
- hosted MCP or hosted trust infrastructure;
- a platform for broad autonomous execution.
