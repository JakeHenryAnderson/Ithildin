# v0.1 Public Preview Review Packet

This packet is the handoff document for a focused v0.1 public-preview release review. It summarizes
what Ithildin currently is, what it deliberately is not, and where an expert reviewer should look
for release blockers.

## Review Request

Assess Ithildin as a local-first governed MCP/tool gateway for AI agents. The intended output is a
release-blocker review, not a broad product brainstorm. Please focus on whether the current v0.1
public-preview boundary is honest, safe enough to publish as local preview, and clearly documented.

## Architecture Summary

Ithildin mediates agent-originated tool calls through a shared governed pipeline:

```text
MCP client / local agent
  -> stdio MCP adapter
  -> trusted tool registry
  -> JSON Schema validation
  -> trusted local principal resolution
  -> resource construction
  -> deny-default policy evaluation
  -> approval workflow when required
  -> scoped executor
  -> redaction
  -> SQLite + hash-chained JSONL audit
```

The FastAPI admin surface and React review console use the same underlying stores and services for
tool listing, approvals, policy preview, audit verification/export, patch proposals, and system
trust status. The MCP adapter stays thin and delegates governance to the shared pipeline.

## Trust Boundaries

| Boundary | v0.1 posture |
| --- | --- |
| Agent/model | Treated as untrusted. No direct OS, shell, Docker, Kubernetes, browser, or arbitrary network access. |
| MCP adapter | Local stdio ingress only. It is not the policy authority. |
| Tool manifests | Trusted local YAML configuration, hash-pinned by `tool-manifests.lock.json`. |
| Principal registry | Trusted local labels for policy and attribution, not production authentication. |
| Policy | Deny-default YAML engine, with optional verified OPA bundle evidence. |
| Executors | Narrow built-ins for workspace reads, git reads, stored patch proposals/apply, and exact-allowlist GET-only HTTP fetch. |
| Admin UI/API | Local admin review surface protected by bearer token. |
| Audit | Local tamper-evident evidence in SQLite and hash-chained JSONL, not immutable custody. |
| Host | Part of the local trusted computing base. Ithildin is not a kernel sandbox. |

## Deferred and Non-Goal Matrix

| Capability | v0.1 stance |
| --- | --- |
| Production identity | Deferred: no OIDC, SAML, SCIM, sessions, or tenant stores. |
| Runtime Postgres | Deferred: SQLite is the only runtime backend. |
| Hosted telemetry | Deferred: OpenTelemetry is opt-in preview metadata only. |
| Remote MCP hosting | Deferred: stdio-only local MCP. |
| Kernel sandbox / EDR / MDM | Non-goal. |
| Shell execution | Not implemented and should remain blocked. |
| Docker socket / Kubernetes tools | Not implemented and should remain blocked. |
| Browser automation | Not implemented and should remain blocked. |
| Arbitrary HTTP methods, headers, bodies | Not implemented; `http.fetch` is GET-only and exact-allowlist-only. |
| Broad filesystem writes/deletes/moves | Not implemented; writes are stored patch proposals plus approval-gated apply only. |
| Plugin SDK/marketplace | Deferred until trust semantics are stronger. |
| Cryptographic signing/notarization | Deferred; hash pinning and tamper-evident local audit only. |

## Release Checklist Snapshot

The required local gate is:

```sh
make release-check
```

This currently runs manifest lock verification, public-preview release guardrails, Python tests,
Python lint, Python typecheck, UI typecheck, docs-site generation, and UI production build.

The secret-free evidence snapshot is:

```sh
make release-evidence
```

It reports the latest commit, dirty-state flag, manifest lock status, tool count, policy status,
principal registry status, storage/telemetry status, local security posture, audit verification when
initialized, and the known deferred boundaries.

Additional optional demo gates:

```sh
make demo-seed
make compose-up
make compose-smoke
make demo-flow
make compose-down
```

Ollama/local model checks skip safely when Ollama or local models are absent:

```sh
make ollama-smoke
make local-model-demo
```

## Reviewer Focus Areas

- Whether the README and release notes describe Ithildin as a local mediation layer without
  overclaiming production security, sandboxing, identity, or immutable audit.
- Whether the approval binding model is sufficient for a public local-preview patch workflow.
- Whether filesystem, patch, and HTTP executor edge cases have obvious bypasses.
- Whether `http.fetch` SSRF controls are appropriately scoped for exact-allowlist GET-only fetch.
- Whether the dev-token, CORS, loopback, Docker socket, and Compose boundaries are release-safe.
- Whether audit evidence language is clear enough that users will not mistake it for tamper-proof
  custody.
- Whether v0.2 priorities should change before public tagging.
