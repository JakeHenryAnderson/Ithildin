# v0.6 Boundary Charter

v0.6 is an external/source-review execution and closure wave for the same narrow v0.1
local-preview runtime boundary. It is not a capability-expansion release, not a production-security
release, and not a broader distribution decision.

## Freeze Point

- Freeze commit: `a43d180c1838afc68dbe496f1b1ea6044e91e386`.
- Frozen input packet: v0.5 handoff packet plus the v0.6 preflight transition note.
- Expected starting state: clean working tree, ten governed tools, external handoff allowed,
  capability expansion blocked, broader public/security-product positioning blocked.
- Required freeze commands:
  - `make release-check`
  - `make v05-review-candidate`
  - `make capability-decision-report`
  - `git status --short`

## v0.6 Purpose

v0.6 should answer one question: what did external/source review actually close?

The wave may prepare review assignment, dispatch focused review packets, normalize incoming external
responses, ingest structured findings, fix or disposition findings, update closure evidence, and
produce go/no-go decisions. It may not infer external closure from internal tests, packet-only
review, or generated evidence.

## Non-Scope

No v0.6 task may add or approve:

- shell execution;
- Docker socket access;
- Kubernetes tools;
- browser automation;
- arbitrary HTTP methods, caller-supplied headers, request bodies, cookies, or broad network access;
- broad filesystem writes, deletes, moves, chmod, archive extraction, or secrets-manager tools;
- plugin SDK or marketplace;
- remote MCP hosting or network MCP transport;
- production identity, OIDC, SAML, SCIM, tenant isolation, hosted sessions, or managed identity;
- runtime Postgres;
- hosted telemetry;
- managed model serving or LLM proxy workflows;
- external audit anchoring, hosted notarization, official hosted supply-chain signing, or
  custody-grade evidence.

## Frozen Surfaces

The following surfaces may not change without a separate boundary decision:

- tool manifests and the ten-tool governed surface;
- MCP transport, which remains local stdio;
- identity model, which remains local principal labels plus local bearer-token admin access;
- runtime storage, which remains SQLite;
- telemetry posture, which remains disabled by default and local-preview only;
- platform-support claims, which remain macOS/Linux local-preview with Windows/WSL unsupported for
  workspace/race security claims.

## Expected Gate State

The healthy v0.6 starting state is:

- external handoff: go;
- capability expansion: no-go;
- broader public/security-product positioning: no-go;
- external/source review rows: pending until real external review outputs exist;
- accepted risks: pending external review until reviewed, fixed, or explicitly accepted-deferred;
- critical/high open findings: none.

Current post-v0.7 patch-apply recheck state differs from the original v0.6 starting state: patch apply is now externally closed for the v0.1 local-preview patch-apply lane; all other source-review lanes remain pending.

## Stop Conditions

Stop autonomous implementation and report status if:

- a critical/high external finding appears;
- a fix requires changing the local-preview boundary;
- a tool manifest changes;
- deferred powers change;
- runtime behavior contradicts the threat model or executor contracts;
- the repo cannot return to a clean verified checkpoint;
- a review response cannot be normalized without ambiguous severity, source access, commit, packet
  hash, or affected file/function evidence.

## Task 181 Acceptance

Task 181 is complete when this charter is wired into the README, review docs, docs site, and release
readiness checks, and the freeze commands still report the expected blocked capability state.
