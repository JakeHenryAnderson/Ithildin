# Ithildin v1.0 Workbench And Evidence Closure

Status: v1.0 local-preview workbench/evidence closure map.

This document ties together the local operator workbench, Agent Run evidence, approval evidence,
audit verification, locally signed evidence, demo artifacts, and review packets for the v1.0 RC
path. It does not add run controls, sandbox orchestration, SIEM adapters, production identity,
runtime Postgres, hosted telemetry, remote MCP, shell/Docker/Kubernetes/browser governed powers,
arbitrary HTTP, broad filesystem writes, compliance automation, or public/security-product claims.

## Operator Questions

The v1.0 local-preview workbench should help a local operator answer:

1. What is the current trust posture?
2. What did the mediated agent do through Ithildin?
3. Which approvals, policy decisions, audit events, diagnostics, and exports correlate with the run?
4. Which evidence artifacts can be handed to a reviewer?
5. Which boundaries remain deferred or explicitly blocked?

## Current Workbench Surfaces

- System Trust panel: local-preview posture, tool count, warning states, manifest/policy evidence.
- Registered Tools panel: current governed tool surface and read-only capability inventory.
- Approval review panel: binding evidence, approve/deny flow, proposal scope, and safe display.
- Agent Runs panel: read-only filters, summary counts, selected-run timeline, grouped evidence, and
  Export Run Evidence.
- Audit surfaces: audit status, audit diagnostics, JSONL export, locally signed export, and
  verification status.
- Demo/evidence packets: workbench packet, live-demo packet, demo evidence packet, signed-evidence
  demo, negative transcripts, and consolidated review candidate.

## Closure Criteria

The workbench/evidence surface is closed for v1.0 local-preview RC when:

- the Agent Run timeline, evidence, and operations readiness gates pass;
- the operator workbench readiness gate passes;
- signed evidence demo generation and verification pass;
- negative transcripts are generated and redaction-safe;
- review candidate generation passes and packet redaction reports `findings: 0`;
- UI tests and UI build pass;
- evidence docs consistently say mediated actions only, local-preview only, locally verifiable only,
  and not production/security/compliance custody.

## Non-Goals

The workbench/evidence surface does not prove OS isolation, host compromise resistance, production
deployment safety, compliance automation, SIEM custody, production identity, remote MCP safety,
container/VM lifecycle control, or activity outside Ithildin-mediated actions.
