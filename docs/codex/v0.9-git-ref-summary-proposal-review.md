# v0.9 git.show.ref_summary Proposal Review

Status: internal xhigh proposal review remediated; implementation remains blocked.

This document records the internal xhigh review of the design-only `git.show.ref_summary` proposal.
It is not external validation and does not authorize manifests, executors, policy rules, MCP
exposure, API behavior, UI behavior, approval behavior, or runtime changes.

## Review Scope

Reviewed material:

- [Capability Proposal: git.show.ref_summary](capability-proposals/git-show-ref-summary.md)
- `scripts/git_ref_summary_proposal_check.py`
- `Makefile`
- README and review-document wiring

The review focused on boundary drift, branch/tag-name privacy, stable hash correlation risk, Git ref
canonicalization, remote/refspec/reflog/revision/pathspec denial coverage, and whether the proposal
is complete enough for later implementation planning.

## Findings And Disposition

| Finding ID | Severity | Disposition | Resolution |
| --- | --- | --- | --- |
| `XH-REFSUMMARY-001` | high | remediated | Raw ref names and current-branch names are now future-only and rejected for the first implementation plan. The proposed response uses response-local opaque `ref_id` values and `is_current_branch` only. |
| `XH-REFSUMMARY-002` | high | remediated | Raw stable `sha256(refname)` values are no longer proposed. The proposal states that stable hashes are not anonymity guarantees and defers any cross-response correlation to a separate reviewed HMAC/salted-digest contract. |
| `XH-REFSUMMARY-003` | medium | remediated for proposal phase | The proposal now requires full-refname validation, Git ref-format round trip, NFC/control-character checks, casefold-conflict handling, symref handling, tag peeling, and commit-object verification before implementation-plan acceptance. |
| `XH-REFSUMMARY-004` | medium | remediated for proposal phase | The proposal now requires strict JSON Schema validation with `additionalProperties: false`, exact enum values, bounded `limit`, and concrete malicious JSON examples for unknown fields and caller-controlled Git inputs. |
| `XH-REFSUMMARY-005` | medium | remediated | README now clarifies that `make v09-design-review-packet` remains the historical `git.show.commit_metadata` packet and that no focused `git.show.ref_summary` handoff packet has been generated yet. |

## Current Decision

`git.show.ref_summary` is acceptable as a design-only candidate after remediation. Implementation
planning remains blocked until a separate implementation-planning document and gate are created.

Required next artifacts before runtime implementation can be considered:

- implementation plan with exact schema, executor, policy fixture, audit, and negative transcript
  details;
- focused source-review handoff packet;
- no-new-powers and tool-surface gate evidence;
- internal xhigh review of the implementation plan;
- explicit implementation decision for this one tool only.

No broader capability implementation or public/security-product positioning is approved by this
proposal review.
