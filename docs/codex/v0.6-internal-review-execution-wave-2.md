# v0.6 Internal Review Execution Wave 2

This report records the internal high-intelligence subagent source-review execution pass for the
remaining v0.6 Wave 2 lanes. It is an internal pressure-test artifact only. It does not close
external/source review rows, does not approve public preview, and does not approve new governed tool
powers.

The run intentionally stopped at findings discovery because several high-severity issues were found.
Per the v0.6 boundary charter, these findings must be remediated or formally dispositioned before
later v0.6 closure, packet, or public-preview decision work continues.

## Remediation Status

The audit/signed-evidence lane has been remediated and converted into fixed structured finding
records:

- `SUB-010`: runtime signed exports now require a clean audit lifecycle and export metadata includes
  lifecycle diagnostics.
- `SUB-011`: offline signed-bundle verification now requires both signature validity and embedded
  audit-chain validity.
- `SUB-012`: SQLite and exported JSONL verification now reject non-object JSON payloads with
  structured invalid results.
- `SUB-013`: plain and signed export endpoints now write safe `audit.exported` events.
- `SUB-014`: the signed-evidence demo verifier now checks recorded artifact path, byte count, and
  SHA-256 metadata.

The policy/registry lane has also been remediated and converted into fixed structured finding
records:

- `SUB-015`: filesystem resources now derive scope through executor-equivalent workspace path
  checks before policy evaluation.
- `SUB-016`: manifest input schemas are validated as JSON Schema at registry load and request-time
  schema errors fail safely.
- `SUB-017`: policy parity now includes resource type/scope expectations and an out-of-scope
  filesystem denial fixture.

The MCP ingress and review-console/admin lanes have also been remediated and converted into fixed
structured finding records:

- `SUB-018`: MCP `tools/call` now denies registered tools that are not exposed through manifest MCP
  metadata.
- `SUB-019`: approval review now checks policy version, matched rules, requesting principal, and
  proposal status alongside the existing binding checks.
- `SUB-020`: the review console now shows principal, workspace, storage, and telemetry posture and
  labels audit-only verification as `Audit chain OK`.
- `SUB-021`: approval mutation routes now reject route/body decision mismatches.

The release automation/evidence gate lane has also been remediated and converted into fixed
structured finding records:

- `SUB-022`: review packet diff gates now recompute listed artifact hashes, detect unlisted
  artifacts, and scan packet contents for secret-like markers.
- `SUB-023`: review bundle `.json` artifacts are now raw JSON, with command transcript metadata in
  adjacent sidecar files.
- `SUB-024`: release evidence validation now binds passing attached release-check transcript claims
  to a readable same-commit transcript and successful return-code evidence.
- `SUB-025`: reviewer finding validators now accept the documented v0.6 namespace forms.
- `SUB-026`: release gates now check generated finding summaries without mutating tracked outputs;
  explicit regeneration uses `make review-findings-summary-write`.

All Wave 2 internal-review findings have been remediated for local-preview review purposes.
External/source review remains pending.
