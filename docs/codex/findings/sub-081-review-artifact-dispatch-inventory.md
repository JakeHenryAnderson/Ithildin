# SUB-081 Review Artifact Dispatch Inventory

- Finding ID: SUB-081
- Severity: medium
- Area: release automation/evidence gates
- Affected files/functions: scripts/reviewer_artifact_manifest.py; scripts/release_guardrails.py; Makefile `review-candidate`
- Claim being tested: The reviewer artifact manifest and release guardrails should reflect all handoff artifacts generated by the current `make review-candidate` path.
- Observed behavior: `make review-candidate` regenerated v0.6 focused dispatch packets, but the reviewer artifact manifest still listed the older artifact set and the release guardrail did not explicitly require the dispatch step in the review-candidate sequence.
- Risk: Reviewers could receive or inspect stale focused dispatch packets without the artifact inventory clearly identifying them as part of the handoff.
- Recommended fix: Add `make v06-review-dispatch-packets` and the dispatch hash/release-automation artifacts to the manifest and guardrail sequence.
- Blocking status: should-fix
- Disposition: fixed
- Verification notes: The reviewer artifact manifest now lists `make v06-review-dispatch-packets`, `dispatch-packet-hashes.json`, and `release-automation.md`. `release_guardrails.REQUIRED_REVIEW_CANDIDATE_STEPS` now includes the dispatch step, and release-readiness tests assert the generated artifact and command coverage. External/source review remains pending.
