## What Changed Since the Last External Packet Review

- Signed audit/export evidence now fails closed on audit lifecycle drift, embeds lifecycle
  diagnostics, and treats embedded audit verification failure as top-level signed-bundle invalidity.
- Audit export endpoints now write `audit.exported` events, and non-object audit payloads fail
  verification structurally instead of crashing verifier paths.
- Filesystem policy resources now derive `in_scope` through executor-equivalent workspace path
  validation before policy/preview decisions.
- Tool manifest input schemas are validated as JSON Schema at registry load.
- Policy parity fixtures now include resource type and scope expectations.
- MCP `tools/call` now enforces manifest MCP exposure instead of relying only on `tools/list`.
- Approval review now mirrors runtime binding checks for policy version, matched rules, requesting
  principal, proposal status, and existing proposal/manifest/schema fields.
- Review console trust posture now includes principal, workspace, storage, telemetry, and clearer
  audit-chain wording.
- Approval mutation routes reject contradictory route/body decisions.
- Review packet diff gates now recompute artifact hashes, detect unlisted packet files, and scan
  generated packet contents for secret-like markers.
- Review bundle `.json` files are raw JSON with sidecar command transcripts.
- Release evidence validation now binds passing transcript claims to a readable same-commit
  transcript and successful return-code evidence.
- Finding namespace validation and finding-summary check-mode are aligned with the documented
  v0.6 review process.
- Policy/registry proxy recheck now covers explicit empty-principal denial, real patch-apply parity
  service wiring, validation-error redaction, pre-policy denial evidence, strict workspace-registry
  behavior, current manifest drift reporting, duplicate YAML key rejection, and dispatch
  traceability.
- MCP ingress recheck now ensures registered-but-unexposed tool calls are denied through shared
  denial evidence instead of silently ending in the adapter.
- Review-console recheck now includes the derived approval-review verdict/checks/reasons in copied
  local evidence so reviewers can compare copied evidence with the visible approve/attention state.
- Release-automation recheck now gives reviewers focused source/test/doc/finding pointers instead
  of broad directory-level dispatch references.
