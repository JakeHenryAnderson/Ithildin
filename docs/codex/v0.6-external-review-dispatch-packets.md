# v0.6 External Review Dispatch Packets

Task 183 adds focused review-packet generation for v0.6 external/source review execution.

## Command

```bash
make v06-review-dispatch-packets
```

Direct form:

```bash
uv run python scripts/external_review_dispatch_packets.py
```

The generated files are ignored local handoff artifacts under:

```text
var/review-packets/v0.6/dispatch/
```

## Packet Set

The generator creates eight focused packets:

- `patch-apply.md`
- `filesystem.md`
- `http-fetch.md`
- `signed-evidence.md`
- `policy-registry.md`
- `mcp-ingress.md`
- `review-console.md`
- `release-automation.md`

It also writes `INDEX.md` and `dispatch-packet-hashes.json`.

Each packet includes a `Dispatch packet payload SHA-256` near the top. That digest covers the packet
payload without the hash-evidence section so reviewers can quote a stable packet-local digest. The
whole-file artifact hash remains in `dispatch-packet-hashes.json`.

## What This Does Not Prove

These packets do not close external review rows, create findings, approve public preview, approve
capability expansion, or prove production readiness. They only make the existing source pointers,
contracts, tests, accepted risks, and command expectations easier for an external reviewer to inspect
area by area.

In short: the dispatch packet set does not close external review rows.

## Required Review Behavior

Each packet asks reviewers to provide reviewer identity, source access level, reviewed commit,
dispatch packet hash, findings in the `EXT-###` convention, blocker status, and a distinction between
documentation risk and implementation risk.

Each packet also includes the same source-access closure rule:

- `source-level` or `packet-and-source` review may support implementation-row closure;
- `packet-only` review may support packet/documentation rows only;
- `docs-only` review may support wording/navigation rows only;
- any closure based on less than source-level access must record the limitation.

Every focused packet ends its expected-command list with `make release-check`.
