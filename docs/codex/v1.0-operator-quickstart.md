# Ithildin v1.0 Operator Quickstart

Status: local-preview operator quickstart for the v1.0 RC path.

This is the front-door path for a local operator or reviewer who wants to run Ithildin without
reading the historical packet trail first. It does not add runtime behavior, governed tools,
approval behavior, sandbox orchestration, SIEM adapters, production identity, remote MCP, hosted
telemetry, runtime Postgres, shell, Docker/Kubernetes/browser governed powers, arbitrary HTTP, broad
filesystem writes, compliance automation, or public/security-product claims.

## What This Starts With

- Current governed tool count: `24`.
- Runtime boundary: local-preview governed MCP/tool gateway.
- Storage: SQLite runtime.
- MCP transport: local stdio.
- Default network posture: `http.fetch` denies unless exact GET allowlist entries are configured.
- Workbench posture: local review console with read-only Agent Run operations, approval evidence,
  audit status, and local evidence exports.

## Zero-To-One Command Path

Run these from the repository root:

```sh
make v1-rc-status-check
make live-demo-preflight
make demo-readiness-summary
make demo-seed
make compose-up
make compose-smoke
make demo-flow
make live-demo-status
make demo-evidence-packet
make workbench-evidence-packet
make review-candidate
make compose-down
```

`make compose-down` is the required cleanup command after any Compose demo.

## Manual Review Steps

After `make compose-up`, open the local review console:

```text
http://127.0.0.1:5173
```

Inspect:

- System Trust and tool count;
- registered tools and manifest evidence;
- approval binding evidence;
- Agent Runs filters, summaries, grouped timeline evidence, and read-only evidence export;
- audit verification and locally signed evidence demo status;
- warning banners for local-preview/development posture.

For MCP stdio demos, use:

```sh
uv run python -m ithildin_mcp_server
```

Then follow the local MCP recipes in [mcp-inspector-recipes.md](mcp-inspector-recipes.md). The MCP
adapter remains an ingress adapter; policy, execution, redaction, approval, and audit stay in the
governed pipeline.

## Evidence Reading Order

After `make review-candidate`, open:

1. `var/review-packets/v3/operator-workbench/WORKBENCH_DEMO_INDEX.md`
2. `var/review-packets/v3/operator-workbench/DEMO_READINESS_SUMMARY.md`
3. `var/review-packets/v3/live-demo/LIVE_DEMO_INDEX.md`
4. `var/review-packets/v3/live-demo/LIVE_DEMO_EVIDENCE_SUMMARY.md`
5. `var/review-packets/v3/demo-evidence/`
6. `var/review-packets/v0.2/GPT-5.5-Pro-consolidated/`

These are ignored/generated evidence artifacts. Do not commit runtime keys, SQLite databases, audit
JSONL files, seeded workspaces, or generated packet contents.

## If Compose Is Unavailable

If Docker Compose is unavailable, skip `make compose-up`, `make compose-smoke`, and `make demo-flow`.
You can still run:

```sh
make live-demo-preflight
make demo-readiness-summary
make demo-workbench
make signed-evidence-demo
make negative-review-transcripts
make review-candidate
```

This proves packaging/readiness consistency, not a live local UI/API run.

## What This Demonstrates

The quickstart demonstrates that a local operator can:

- check local-preview posture;
- seed a safe ignored demo workspace;
- run a local UI/API demo when Compose is available;
- observe mediated actions through Ithildin-controlled surfaces;
- inspect approval, audit, Agent Run, and evidence-export artifacts;
- generate a redaction-scanned review candidate packet;
- shut the demo down cleanly.

## What This Does Not Demonstrate

This quickstart does not prove production deployment safety, OS isolation, host compromise
resistance, custody-grade audit, SIEM retention, compliance automation, production identity,
remote MCP safety, broad capability approval, or activity outside Ithildin-mediated actions.
