# v0.3 Boundary Decision

Decision date: 2026-05-30

Decision: Ithildin is ready for external/source review handoff as a v0.3-prep review candidate for
the v0.1 local-preview runtime boundary.

This is not a decision to broaden tool powers, claim production readiness, market Ithildin as a
security product, or treat local evidence as immutable/custody-grade proof.

## Basis

Tasks 085-112 built review automation, adversarial tests, fail-closed validation suites, evidence
schema validation, packet generation, review intake, and boundary documentation without adding new
governed tool powers.

The handoff is supported by:

- `make release-check`;
- `make review-candidate`;
- `make internal-review-packet`;
- source-review closure matrix v2;
- structured reviewer finding intake;
- executor contract set;
- manifest/principal/workspace/audit fail-closed suites;
- SUB-001 finding record showing internal remediation and pending external review;
- v0.3 external review packet and prompt.

## Allowed Next Step

Send the v0.3-prep consolidated packet and relevant source files to GPT 5.5 Pro / Very High or a
human expert for external/source review.

## Not Allowed Yet

Do not add:

- shell execution;
- Docker socket access;
- Kubernetes tools;
- browser automation;
- arbitrary HTTP methods, caller-supplied headers, request bodies, cookies, or broad network access;
- broad filesystem writes, deletes, moves, chmod, archive extraction, or secrets-manager tools;
- plugin SDK or marketplace;
- remote hosted MCP gateway;
- production identity integrations;
- runtime Postgres;
- hosted telemetry collectors;
- managed model serving or LLM proxy workflows.

## Required Before Any Boundary Expansion

Before any new tool-power class or production/security-product positioning:

1. External/source review must inspect the current boundary.
2. Blocking external findings must be fixed or explicitly accepted as deferred with rationale.
3. The source-review closure matrix must be updated through
   [external-review-intake-and-closure.md](external-review-intake-and-closure.md).
4. `make reviewer-findings-check`, `make release-check`, and `make review-candidate` must pass.
5. A new boundary decision must state what changed and why the expansion is acceptable.

## Current Positioning

Ithildin remains a local-first governed MCP/tool gateway. It mediates narrow local tools through
trusted manifests, schema validation, local principal labels, named workspaces, deny-default policy,
approval, scoped executors, redaction, and local tamper-evident audit.

It is not a sandbox, production identity system, SIEM, EDR/MDM product, hosted MCP platform,
compliance audit system, immutable evidence store, or production control plane.
