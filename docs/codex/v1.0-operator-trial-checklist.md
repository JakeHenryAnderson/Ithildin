# Ithildin v1.0 Operator Trial Checklist

Status: Local-v1 operator trial companion.

This checklist records one attempt at the
[Ithildin Local v1.0 Golden Path](local-v1-golden-path.md). The golden path owns the executable
steps, expected outputs, stop conditions, two-leg truth boundary, and cleanup sequence; do not
duplicate or reorder it here. This checklist does not
add runtime behavior, governed tools, approval behavior, sandbox orchestration, SIEM adapters,
production identity, remote MCP, hosted telemetry, runtime Postgres, shell, Docker/Kubernetes/browser
governed powers, arbitrary HTTP, broad filesystem writes, compliance automation, or
public/security-product claims.

## Trial Metadata

Record these values before starting:

| Field | Expected value or note |
| --- | --- |
| Repository root | Local Ithildin checkout. |
| Commit under trial | `git rev-parse HEAD`. |
| Dirty state | `git status --short` is empty before final handoff. |
| Governed tool count | `24`. |
| Golden path source | `docs/codex/local-v1-golden-path.md`. |
| Operator mode | Local-v1 local preview only. |
| Normal Compose leg | completed, skipped, or failed-safe. |
| Real Hermes evidence leg | existing evidence validated, deliberately reproduced, unavailable, or failed. |
| Synthetic Node/Mission evidence leg | validated, unavailable, or failed. |

## Required Preflight

Run:

```sh
make local-v1-golden-path-check
make live-demo-preflight
make demo-readiness-summary
```

Expected result:

- each command exits successfully;
- tool count remains `24`;
- an existing `.env` has no group/world permission bits;
- the real Hermes MCP and synthetic Node/Mission Command legs remain separate;
- `MCC-007` implementation remains unauthorized;
- public/security-product positioning remains blocked;
- any missing Docker Compose support is recorded as optional/manual, not as production failure.

## Golden Path Record

Follow the golden path in order. Record:

- whether the normal API/UI opened at `http://127.0.0.1:5173`;
- the allowed read, policy previews, and automatically completed approval lifecycle observed from
  `make demo-flow`;
- the bounded evidence exports inspected;
- whether retained historical Hermes evidence validated or reproduction was deliberately run,
  including that its legacy checker does not prove current-candidate freshness;
- whether retained synthetic Node evidence validated without treating its legacy checkers as
  current-candidate provenance;
- whether the Node release-artifact check's recorded-commit binding was kept distinct from current
  worktree cleanliness;
- whether the MCC-006 evidence checker established its exact-current-clean-candidate binding;
- any unavailable/stale evidence without silently reproducing it;
- whether `make compose-down` and, when applicable, `make hermes-poc-stop` completed;
- final `git status --short` output and any retained ignored evidence.

The ordinary operator trial does **not** run the historical full governance gate or qualify a
candidate. `make release-check` is reserved for deliberate full-integrity checkpoints.
`make local-v1-candidate-check` is reserved for the later frozen-candidate milestone and remains
fail-closed while required outcomes are incomplete.

## Trial Pass Criteria

An operator trial passes for the v1.0 local-preview boundary when:

- required preflight passes;
- the golden path is followed without joining its two evidence legs;
- the normal Compose leg is exercised and cleaned up, or explicitly recorded as skipped;
- existing evidence is validated or recorded honestly as unavailable;
- bounded exports contain no production credential or unredacted secret;
- no critical/high local-preview finding opens;
- no document or UI text claims production readiness, sandbox isolation, SIEM custody, compliance
  automation, production identity, or public/security-product positioning.

## Trial Fail Criteria

Stop and record failure if:

- the golden-path validator, preflight, or an attempted live step fails;
- a bounded export exposes a credential or unredacted secret;
- a critical/high finding opens;
- Compose starts but cannot be cleaned up with `make compose-down`;
- optional Hermes reproduction starts but cannot be cleaned up with `make hermes-poc-stop`;
- the trial requires a new governed tool, `MCC-007` implementation, generic process control,
  sandbox orchestration, trusted-host promotion, Mission Control execution authority, or a
  product-boundary change.

## What The Trial Does Not Prove

This checklist does not prove production deployment safety, OS isolation, host compromise
resistance, custody-grade audit, SIEM retention, compliance automation, production identity,
remote MCP safety, broad capability approval, live VM/container control, Mission Control execution,
trusted-host promotion, or activity outside Ithildin-mediated actions.
It does not prove a real Hermes-through-Node mission, qualify a release candidate, complete human
UAT, or accept Local v1.0.
