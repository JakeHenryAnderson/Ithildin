# Ithildin v1.0 Operator Trial Checklist

Status: local-preview operator trial checklist for the v1.0 RC path.

This checklist turns the v1.0 operator quickstart into a repeatable local trial record. It does not
add runtime behavior, governed tools, approval behavior, sandbox orchestration, SIEM adapters,
production identity, remote MCP, hosted telemetry, runtime Postgres, shell, Docker/Kubernetes/browser
governed powers, arbitrary HTTP, broad filesystem writes, compliance automation, or
public/security-product claims.

## Trial Metadata

Record these values before starting:

| Field | Expected value or note |
| --- | --- |
| Repository root | Local Ithildin checkout. |
| Commit under trial | `git rev-parse HEAD`. |
| Dirty state | `git status --short` is empty before final handoff. |
| Governed tool count | `24`. |
| Latest implemented tool | `sandbox.artifact.write_text`. |
| Selected next capability | `not selected`. |
| Operator mode | local-preview only. |
| Compose mode | available, skipped, or failed-safe. |

## Required Preflight

Run:

```sh
make v1-rc-status-check
make v1-progress-assessment
make live-demo-preflight
make demo-readiness-summary
```

Expected result:

- each command exits successfully;
- tool count remains `24`;
- capability expansion remains blocked;
- public/security-product positioning remains blocked;
- any missing Docker Compose support is recorded as optional/manual, not as production failure.

## Optional Compose Path

Run only when Docker Compose is available and the operator wants the local API/UI demo:

```sh
make demo-seed
make compose-up
make compose-smoke
```

Manual checks:

- review console opens at `http://127.0.0.1:5173`;
- System Trust shows local-preview posture and tool count;
- registered tools and manifest evidence are visible;
- Agent Runs panel can show filters, summaries, timeline evidence, and read-only evidence export;
- approval, audit, and signed-evidence surfaces show warnings and safe metadata only.

Cleanup requirement:

```sh
make compose-down
```

If Compose is unavailable, skip this section and record `Compose skipped`. Do not treat skipped
Compose as proof that the live API/UI path was exercised.

## Non-Compose Evidence Path

Run:

```sh
make demo-workbench
make signed-evidence-demo
make negative-review-transcripts
make live-demo-status
make demo-evidence-packet
make workbench-evidence-packet
```

Expected result:

- generated packets are secret-free ignored artifacts;
- live-demo status records whether API/UI are reachable;
- signed evidence demo remains non-production fixture evidence;
- negative transcripts remain denial/evidence examples only.

## Final Handoff Gate

Run:

```sh
make release-check
make review-candidate
git status --short
```

Expected result:

- `make release-check` passes;
- `make review-candidate` passes;
- packet redaction scan reports `findings: 0`;
- `git status --short` is empty after committed source changes;
- v1.0 RC packet exists at `var/review-packets/v1.0/rc/`;
- enterprise next-review handoff exists at
  `var/review-packets/v3/enterprise-next-review-handoff/NEXT_ENTERPRISE_REVIEW_HANDOFF.md`.

## Trial Pass Criteria

An operator trial passes for the v1.0 local-preview boundary when:

- required preflight passes;
- either the optional Compose path is exercised and cleaned up, or it is explicitly skipped;
- non-Compose evidence path passes;
- final handoff gate passes;
- no critical/high local-preview finding opens;
- no document or UI text claims production readiness, sandbox isolation, SIEM custody, compliance
  automation, production identity, or public/security-product positioning.

## Trial Fail Criteria

Stop and record failure if:

- `make release-check` fails;
- `make review-candidate` fails;
- packet redaction findings are nonzero;
- a critical/high finding opens;
- Compose starts but cannot be cleaned up with `make compose-down`;
- the trial requires a new governed tool, new runtime behavior, sandbox orchestration, trusted-host
  promotion, Mission Control execution authority, or product-boundary change.

## What The Trial Does Not Prove

This checklist does not prove production deployment safety, OS isolation, host compromise
resistance, custody-grade audit, SIEM retention, compliance automation, production identity,
remote MCP safety, broad capability approval, live VM/container control, Mission Control execution,
trusted-host promotion, or activity outside Ithildin-mediated actions.
