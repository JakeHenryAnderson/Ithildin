# Mission Control Enterprise Status Import Contract

Status: display-only contract for future Mission Control enterprise status imports.

`make mission-control-enterprise-status-import-check` validates how a future Mission Control
display-only importer should treat the Ithildin enterprise status export generated by
`make enterprise-status-export`. This is a fixture/import contract only. It does not add Mission
Control runtime behavior, callbacks into Ithildin, API polling, MCP behavior, approvals, audit
authority transfer, local model invocation, VM/container lifecycle management, sandbox
orchestration, trusted-host promotion, SIEM adapters, production identity, runtime Postgres, hosted
telemetry, shell, Docker/Kubernetes/browser governed powers, arbitrary HTTP, broad filesystem
writes, compliance automation, or public/security-product claims.

## Source Artifact

The importer target is the generated JSON artifact:

```text
var/review-packets/v3/enterprise-status-export/enterprise-status-export.json
```

The source artifact must identify itself as:

- `artifact_type: ithildin.enterprise_status_export`
- `status: display_only`
- `schema_version: 1`

Mission Control may treat the artifact as local dashboard input only after validating the schema and
all blocked-authority flags. Ithildin remains the execution, policy, approval, audit, and lane closure authority.

`make mission-control-enterprise-status-fixtures` generates one valid payload and negative
`MC-STATUS-NEG-###` payloads for future Mission Control importer tests. The fixture pack remains
local display/import evidence and does not call Mission Control.
`make mission-control-enterprise-status-acceptance-matrix-check` validates the matching
fixture-to-importer acceptance matrix.
`make mission-control-enterprise-status-reference-validator` validates the same fixture pack as an
Ithildin-side display-only oracle for future Mission Control importer tests.

## Allowed Display Fields

A future Mission Control importer may display only these status fields:

- schema version, artifact type, display-only status, commit, and dirty state;
- tool count, selected capability, response count, closure count, and enterprise gap count;
- recommended enterprise review IDs, the recommended next enterprise review, the current next action,
  safe action commands, and post-send receipt/response breadcrumbs from
  `next_after_send_commands`;
- display-only handoff artifact labels and relative artifact paths;
- progress bands, review lane labels/statuses, packet path labels, send package/send-session record
  pointers, and packet handoff readiness;
- blocked-authority flags and safe warning chips.

The importer must not display prompts, file contents, diffs, response bodies, raw host secrets,
tokens, private keys, environment values, raw stack traces, arbitrary JSON subtrees, or unreviewed
runtime payloads.

The `handoff_artifacts` field is display-only. Mission Control may render labels and relative
paths as copyable text, but must not execute commands, open host paths with elevated authority,
poll Ithildin, or infer that an artifact path grants runtime permission.

The `action_commands` and `next_after_send_commands` fields are display-only copy guidance for a
human operator. Mission Control must not convert either field into executable controls, background
jobs, polling behavior, reviewer-response normalization, or lane-closure authority.

The `packet_paths.enterprise_review_send_package` and
`packet_paths.enterprise_review_send_session_record` fields are display-only pointers. Mission
Control may show them as operator guidance, but must not treat the package as review evidence or the
session record as proof that a review occurred.

## Required Refusals

The future importer must reject or quarantine artifacts that:

- have any artifact type other than `ithildin.enterprise_status_export`;
- have any status other than `display_only`;
- report Mission Control as a runtime, execution, policy, approval, or audit authority;
- report live VM/container inspection, sandbox orchestration, trusted-host promotion, SIEM adapter
  behavior, compliance automation, public/security-product positioning, new governed tool powers, or
  runtime changes as allowed;
- omit required safe status fields or make lane closure claims without normalized external
  responses.

## Boundary

This contract does not approve Mission Control enterprise status importer implementation. It only
defines the Ithildin-side display-only acceptance posture that a future Mission Control importer can
test against.

Mission Control may display this artifact as non-authoritative status. Mission Control must not use it to execute work. Mission Control must not use it to poll Ithildin. Mission Control must not use it to invoke a model. Mission Control must not use it to inspect a VM/container. Mission Control must not use it to promote host artifacts. Mission Control must not use it to close review lanes. Mission Control must not use it to approve runtime behavior. Mission Control must not use it to grant new tool powers.
