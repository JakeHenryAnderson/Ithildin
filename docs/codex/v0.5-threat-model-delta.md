# v0.5 Threat Model Delta

Task 175 records the threat-model delta for Task 152 through Task 175. v0.5 remains a review-closure and
capability-decision preparation wave over the v0.1 local-preview runtime boundary.

## What Changed

- Review gates now make capability expansion, source-review closure, accepted risks, and tool-surface drift
  visible as executable checks.
- Source-review artifacts now include runbooks, source pointers, transcript packet templates, reviewer
  artifact manifests, and external response intake templates.
- Accepted risks are explicit and machine-checked.

## What Did Not Change

- No new governed tool powers were added.
- The tool surface remains the existing ten local-preview tools.
- External/source review remains pending.
- Accepted risks remain local-preview only.
- Capability expansion remains blocked.
- Ithildin is still not a sandbox.
- Ithildin still provides local principal labels, not production identity.
- Locally signed evidence is still local operator evidence, not external notarization.

## Residual Threats

- The local host, local admin, manifests, policies, workspaces, and Ithildin process remain part of the
  trusted computing base.
- Audit evidence remains locally verifiable and tamper-evident, not immutable custody.
- Patch apply, filesystem behavior, and HTTP fetch remain the highest-priority source-review surfaces.
- Broader distribution and any future powerful tool class require external/source review and a separate
  capability decision.
