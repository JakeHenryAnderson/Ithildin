# v0.4 Threat Model Refresh

Task 148 refreshes the local-preview threat model after the v0.4 review-closure and evidence
maturity work. It does not introduce new tool powers or stronger production claims.

## Current Boundary

Ithildin remains a local-preview governed MCP/tool gateway. The local host, local admin, trusted
manifests, trusted policies, trusted workspace/principal registries, local signing keys, and the
Ithildin process are part of the trusted computing base.

The agent/model remains untrusted. The MCP adapter remains an ingress adapter. Executors remain
narrow application-level mediators, not OS sandboxes.

## v0.4 Changes That Affect Threat Evidence

- Review-run manifests, finding aggregation, and closure-matrix v3 make review status explicit.
- Patch apply failure evidence records incomplete apply attempts for read-only diagnostics.
- Filesystem support evidence limits workspace/race security claims to macOS and Linux local-preview
  profiles.
- HTTP canonicalization corpus and executor contract v2 document exact allowlist, redirect, DNS/IP,
  proxy, timeout, and size-limit behavior.
- Redaction evidence boundary clarifies that redaction is best-effort leak reduction.
- Demo scenario pack and review docs index make reviewer handoff easier to reproduce.

## Accepted Local-Preview Risks

- A compromised local host or local admin can bypass or tamper with local controls.
- SQLite/JSONL audit remains local tamper-evident evidence, not custody-grade immutable storage.
- Local Ed25519 signatures prove local key possession only; they are not external notarization or
  hosted supply-chain signing.
- Redaction can miss unknown secret formats and must not be treated as a guarantee.
- Filesystem/race claims are not security-supported on Windows or WSL.
- Demo transcripts and scenario packs are local observations, not external implementation review.

## Stop Conditions

Autonomous implementation must stop before proceeding if:

- a critical or high internal/external finding is open;
- source behavior contradicts this threat refresh, the main threat model, or the v0.4 charter;
- a task requires shell, Docker socket, Kubernetes, browser automation, arbitrary HTTP, broad
  filesystem writes, production identity, runtime Postgres, hosted telemetry, remote MCP, or a
  plugin SDK;
- release evidence, review packet hashes, signing evidence, or audit evidence become ambiguous.

## Reviewer Question

For v0.4 external/source review, ask whether the documented local-preview threat boundary matches
the implementation evidence and whether any accepted risk should block future capability expansion.
