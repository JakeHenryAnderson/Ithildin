# v0.8 Public-Preview Risk Review

Wave 3 records the v0.8 public-preview claims decision. It is a product-risk decision, not a new
runtime capability and not a production-security approval.

## Decision Table

| Question | Decision | Conditions |
| --- | --- | --- |
| Continued local-preview development | go | Keep the v0.1 local-preview runtime boundary unchanged. |
| Limited technical-preview sharing | conditional_go | Share only with the warning packet, local-preview wording, and no-new-powers guardrails. |
| Broad public/security-product positioning | no_go | Do not market Ithildin as a security product, sandbox, EDR/MDM, SIEM, compliance tool, hosted MCP, or production control plane. |
| Production/security/compliance positioning | no_go | Production identity, custody-grade audit, runtime Postgres, hosted telemetry, remote MCP, and notarization remain deferred. |

## Audience Boundary

The acceptable audience for limited technical-preview sharing is security-conscious developers,
local AI-tooling builders, and reviewers who can understand the local-preview trust model. The
non-target audience is production operators seeking features that are not supported: enterprise
identity, compliance evidence, endpoint protection, hosted control planes, remote MCP, or broad
autonomous execution.

## Required Public-Preview Warnings

Any shared packet, README, quickstart, release note, or UI status text must preserve these warnings:

- local preview only;
- not a sandbox;
- not production security;
- not production identity;
- not immutable or custody-grade evidence;
- not external notarization;
- not hosted MCP;
- redaction is best-effort leak reduction;
- macOS/Linux local-preview filesystem support only;
- no shell, Docker socket, Kubernetes, browser automation, arbitrary HTTP, broad filesystem writes,
  plugin SDK, hosted telemetry, runtime Postgres, or new governed tool powers.
- no new governed tool powers.

## Support Boundary

Operators may ask for help running the local demo, inspecting review packets, understanding
governed-tool evidence, and reproducing local-preview tests. Unsupported requests include
production deployment, not enterprise identity integration, compliance attestation, remote MCP
hosting, new executor powers, secrets-manager access, or hosted telemetry.

## Go/No-Go

- Local-preview development: `go`.
- Limited technical-preview sharing: `conditional_go`.
- Public/security-product positioning: `no_go`.
- Capability implementation: `no_go`.
- Capability design: still pending Wave 4.
