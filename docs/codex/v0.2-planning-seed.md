# v0.2 Planning Seed

This is the v0.2 trust-evidence roadmap after the v0.1 public-preview review. Ithildin is now a
v0.2 review candidate for the v0.1 local-preview runtime boundary. The current v0.2 hardening wave
is implemented through Task 080 and is ready for external/code review.

## Planning Principle

v0.2 strengthened trust semantics before adding new governed powers. The highest-value work was not
more tools; it was clearer evidence, stronger approval equivalence, better review workflows, and
more testable policy.

## Proposed Theme

**Trust Evidence v2:** make Ithildin easier to review, harder to misconfigure, and more precise
about what was approved, executed, and exported.

## Next After v0.2

- Run an external/code review against the generated review bundle,
  [v0.2-review-response-and-rc-cleanup.md](v0.2-review-response-and-rc-cleanup.md),
  [v0.2-review-packet.md](v0.2-review-packet.md),
  [internal-source-review-pass-1.md](internal-source-review-pass-1.md), the release packet output,
  and the security matrix before expanding capabilities.
- Use the Task 080 filesystem executor contract and capability check as external-review input before
  broader distribution or broader write powers.
- Decide whether locally signed evidence should become signed release artifacts or external
  anchoring; do not imply notarization until a real trust root exists.
- Decide production identity and runtime storage direction before any remote MCP, hosted control
  plane, plugin SDK, or materially broader tool powers.
- Keep new tool powers deferred until approval binding, evidence contracts, and executor contracts
  have had review outside this development loop.

## Keep Deferred

- Shell execution.
- Docker socket access.
- Kubernetes tools.
- Browser automation.
- Arbitrary HTTP methods, caller-supplied headers, request bodies, cookies, or broad network access.
- Broad filesystem writes, deletes, moves, chmod, archive extraction, or secrets-manager tools.
- Plugin SDK or marketplace.
- Remote hosted MCP gateway.
- Production identity integrations: OIDC, SAML, SCIM, hosted sessions, tenant stores.
- Runtime Postgres unless concurrency or multi-user requirements become unavoidable.
- Hosted telemetry collectors.
- Managed model serving or LLM proxy workflows.

## Review Questions For External v0.2 Review

- Are the signed local evidence formats useful enough without overclaiming notarization or custody?
- Are policy preview, policy test fixtures, and runtime policy evidence comparable enough for review?
- Are approval binding checks, replay prevention, and patch apply revalidation sufficiently explicit?
- Are the remaining deferred boundaries clear enough to prevent accidental scope expansion?
- Should OPA become canonical after v0.2, or remain optional evidence/prototype support?
