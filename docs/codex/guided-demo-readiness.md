# Guided Demo Readiness

Status: release-readiness gate.

The guided local demo path turns the operator workbench evidence into a single repeatable
sequence. It is product-demo polish only. It does not add governed tools, run controls, sandbox
orchestration, SIEM adapters, production identity, remote MCP, hosted telemetry, shell access,
Docker/Kubernetes powers, arbitrary HTTP, broad filesystem writes, or plugin SDK behavior.

## Commands

- `make guided-demo` runs the non-service-starting guided demo path.
- `make demo-state-report` writes `DEMO_STATE_REPORT.md` with current demo state.
- `make demo-reset-guide` writes `DEMO_RESET_GUIDE.md` with read-only reset guidance.
- `make demo-flow-readiness` validates the result/reset/UI demo label wiring.
- `make demo-flow-result-check`, `make demo-evidence-packet`, and
  `make demo-evidence-readiness` validate and package optional mediated demo evidence.
- `make guided-demo-readiness` validates command, docs, UI, packet, and no-new-powers wiring.

`make guided-demo` runs preflight, seeds the ignored demo workspace, writes the demo state report,
generates the operator demo guide, refreshes the workbench smoke transcript, and rebuilds the
operator workbench evidence packet. It does not start Compose, does not stop services, does not call
governed tools, approve actions, run browser automation, or manage sandbox lifecycle.

Boundary shorthand: `make guided-demo` does not call governed tools.

## Operator Path

1. Run `make guided-demo`.
2. Inspect `DEMO_STATE_REPORT.md` for ready, missing, warning, and next-command state.
3. Start local services manually only if intended: `make compose-up && make compose-smoke`.
4. Launch the local MCP server with `uv run python -m ithildin_mcp_server`.
5. Run `make demo-flow` to create a mediated local demo run.
6. Inspect `DEMO_FLOW_RESULT.md` for proposal, approval, audit, and candidate run IDs.
7. Open `http://127.0.0.1:5173` and inspect Agent Runs.
8. Select a run and use Export Run Evidence.
9. Run `make demo-reset-guide` if the demo needs repeat/recovery guidance.
10. Run `make compose-down` if Compose was started.

## Gate

`make guided-demo-readiness` checks:

- `make guided-demo`, `make demo-state-report`, and `make guided-demo-readiness` exist;
- `make guided-demo-readiness` is in `release-check`;
- `make guided-demo` is in `review-candidate`;
- `DEMO_STATE_REPORT.md` and `GUIDED_DEMO_TRANSCRIPT.md` are generated by the guided path;
- README and the reproduction map mention the guided demo commands;
- the review console explains the empty Agent Runs state and export availability;
- tool count remains `20`;
- no-new-powers and tool-surface guardrails still pass.

## Boundary

The guided demo path is not a runtime fixture loader, hosted demo runner, sandbox controller,
SIEM adapter, compliance workflow, browser automation harness, or production deployment proof. It
does not prove OS isolation, host compromise resistance, custody-grade audit, external
notarization, or activity outside Ithildin-mediated actions.
