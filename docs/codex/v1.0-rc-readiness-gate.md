# Ithildin v1.0 RC Readiness Gate

Status: v1.0 local-preview release-candidate readiness gate.

This gate is the v1.0 RC umbrella for local-preview readiness. It does not add runtime behavior,
new governed tools, run controls, sandbox orchestration, SIEM adapters, production identity,
runtime Postgres, hosted telemetry, remote MCP, shell/Docker/Kubernetes/browser powers, arbitrary
HTTP, broad filesystem writes, plugin SDK behavior, compliance automation, or public/security-product
claims.

## Gate Inputs

The gate composes these local reports:

- `make v1-rc-status-check`
- `make v1-rc-roadmap-check`
- `make v1-operator-quickstart-check`
- `make v1-workbench-evidence-check`
- `make v1-assurance-closure-check`
- `make agent-workflow-check`
- `make tool-surface-invariant-gate`
- `make no-new-powers-guardrail`
- `make read-only-capability-inventory-gate`
- `make next-capability-readiness`
- `make capability-decision-report`
- `make packet-redaction-scan`

## RC Interpretation

The gate can pass only when:

- governed tool count remains `23`;
- latest implemented tool remains `project.risk.summary`;
- no next capability is selected;
- capability expansion remains blocked;
- public/security-product positioning remains blocked;
- no new power classes are introduced;
- packet redaction reports zero findings;
- local-preview evidence docs are wired into docs-site and review-doc metadata;
- external/source-review pending rows remain visible instead of being called closed.

## Packet

`make v1-rc-packet` generates a compact ignored packet under `var/review-packets/v1.0/rc/` with the
current status, operator quickstart, workbench/evidence map, assurance map, command checklist, and
artifact hashes. The packet is for local handoff convenience only; it is not notarization,
production approval, or external source-review closure.

## Required Commands

```sh
make v1-rc-readiness
make v1-rc-packet
make release-check
make review-candidate
```

## Non-Goals

This gate does not prove implementation correctness, production deployment readiness, compliance
readiness, OS isolation, host compromise resistance, custody-grade audit, external notarization, or
activity outside Ithildin-mediated actions.
