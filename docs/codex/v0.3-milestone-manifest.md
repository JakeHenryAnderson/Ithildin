# v0.3 Milestone Manifest

This manifest records the planned v0.3-prep implementation-assurance band from Tasks 085-112. In
practice, this is a review and hardening wave over the same narrow v0.1 local-preview runtime
boundary, not a new capability release.

The machine-readable companion is
[v0.3-milestone-manifest.json](v0.3-milestone-manifest.json). That JSON file is the source of truth
for task IDs, wave groupings, commit labels, stop conditions, subagent policy, and deferred
boundaries.

## Operating Rule

Each milestone must be independently committed. Each wave must pass its gate before the next wave
starts. Internal high-intelligence subagents may pressure-test implementation and evidence, but they
cannot close external source-review rows or approve new product-boundary claims.

Stop and report status if the same blocking failure repeats three times, if a trust-boundary
regression appears, if a critical/high internal finding appears, if the work requires changing the
product boundary, or if release/audit/manifest/signing evidence becomes ambiguous.
