# Mission Control Handoff Schema Contract

Status: design-only Ithildin-side schema contract for Mission Control display/import.

This contract defines the first stable handoff shape that Mission Control may import or display from
Ithildin evidence packets. It does not add runtime behavior, API callbacks, MCP transports, Mission
Control execution behavior, local model invocation, VM/container lifecycle management, sandbox
orchestration, trusted-host promotion, SIEM adapters, production IAM, runtime Postgres, hosted
telemetry, shell, Docker/Kubernetes/browser governed powers, arbitrary HTTP, broad filesystem writes,
compliance automation, or public/security-product claims.

The current concrete seed is generated by:

```sh
make hello-world-mission-control-handoff
make hello-world-mission-control-handoff-check
```

Generated payload:

```text
var/review-packets/v3/hello-world-mission-control-handoff/mission-control-handoff.json
```

Concrete importer fixtures are generated by:

```sh
make mission-control-handoff-fixture-pack
make mission-control-handoff-fixture-pack-check
```

Generated fixture pack:

```text
var/review-packets/v3/mission-control-handoff-fixtures/
```

## Contract Purpose

The contract lets Mission Control act as an operator display surface for Ithildin evidence while
keeping authority split cleanly:

- Ithildin remains the governed gateway, policy authority, approval authority, execution mediator,
  and audit evidence source.
- Mission Control may import or display labels, IDs, hashes, warning states, and relative evidence
  links.
- Mission Control must not infer execution, policy, approval, audit, local-model, sandbox, VM, or
  trusted-host promotion authority from this packet.

## Top-Level Required Fields

Every handoff payload using this contract must include:

| Field | Required value or shape | Notes |
| --- | --- | --- |
| `schema_version` | `"1"` | Increment for incompatible changes. |
| `handoff_type` | stable string beginning with `mission_control.` | Identifies the packet family. |
| `status` | `"metadata_only"` | No live integration state. |
| `generated_at` | ISO-8601 timestamp string | Evidence timestamp only. |
| `mission_control_runtime_behavior` | `false` | Required visible warning source. |
| `mission_control_authority` | `"display_and_operator_review_only"` | No executor/policy authority. |
| `ithildin_remains_policy_authority` | `true` | Must stay true. |
| `local_llm_runtime_behavior` | `false` | No local model execution in this handoff. |
| `real_vm_or_container_started` | `false` | No VM/container lifecycle action. |
| `sandbox_orchestration_performed` | `false` | No sandbox orchestration. |
| `shell_execution_performed` | `false` | No shell execution. |
| `host_promotion_performed` | `false` | No trusted-host promotion. |
| `tool_count` | integer | Must match current Ithildin evidence. |
| `mission` | object | Display labels and review state only. |
| `model_plan` | object | Model/client labels and content hash only. |
| `ithildin_evidence` | object | Governed request, approval, execution, artifact, and audit metadata. |
| `attachments` | list of objects | Relative evidence links only. |
| `display_contract` | object | Explicit display allowlist and hidden-field denylist. |
| `boundaries` | object | Required no-authority/no-content booleans. |

## Display Allowlist

Mission Control may display these categories:

- mission ID and operator intent label;
- local-preview status labels;
- model/client label as metadata only;
- run/request/approval/execution status labels;
- request ID and approval ID;
- artifact label, byte count, and SHA-256 hashes;
- audit verification status, audit event count, and audit head hash;
- relative attachment paths and artifact hashes;
- warning chips derived from the required false booleans.

Mission Control must keep the warning chips visible whenever it displays this handoff:

- `local_preview_only`;
- `mission_control_metadata_only`;
- `local_llm_not_invoked`;
- `vm_not_started`;
- `host_promotion_not_performed`.

## Hidden-Field Denylist

The handoff payload and Mission Control display must not expose:

- file contents;
- raw host paths;
- raw prompts;
- chain-of-thought;
- private keys;
- tokens;
- environment values;
- dependency names;
- package script values;
- response bodies;
- raw sandbox internals;
- production/compliance conclusions.

## Attachment Rules

Attachment paths must be packet-relative paths. They must not be absolute paths, parent-directory
traversals, URLs, shell commands, or runtime instructions.

Attachment entries may include:

- `label`;
- `path`;
- `purpose`;
- optional digest fields in future schema versions.

## Negative Cases

Mission Control import/display planning must reject or visibly warn on:

- missing `schema_version`;
- unsupported `schema_version`;
- `status` other than `metadata_only`;
- any required authority or runtime boolean set to `true`;
- missing `ithildin_remains_policy_authority: true`;
- missing `display_contract`;
- missing hidden-field denylist;
- missing warning chips;
- absolute attachment paths;
- parent-directory traversal in attachment paths;
- packet fields claiming Mission Control execution, policy authority, approval authority, audit
  authority, sandbox/VM lifecycle control, local model invocation, or trusted-host promotion.

## Current Implementation Boundary

This contract is an Ithildin-side handoff schema contract only. Runtime implementation remains
blocked until a later explicit implementation plan, implementation gate, source-review handoff, and
release/readiness update. Mission Control-side work should consume this as a display/import contract,
not as authorization to call back into Ithildin or perform governed actions.
