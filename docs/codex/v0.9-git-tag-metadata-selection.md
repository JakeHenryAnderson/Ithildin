# v0.9 git.show.tag_metadata Selection

Status: design-only candidate selection. This document records `git.show.tag_metadata` as the
next bounded read-only Git metadata candidate after `git.show.ref_summary`.

This selection does not add a manifest, executor, policy rule, MCP exposure, API behavior, UI
behavior, approval behavior, audit behavior, or runtime behavior. Implementation remains blocked
until proposal, implementation planning, implementation decision, source-review handoff, manifest
lock update, policy/parity coverage, and release/readiness gates are complete.

## Decision

| Candidate | Usefulness | Main risk | Decision |
| --- | --- | --- | --- |
| `git.show.tag_metadata` | Helps an agent orient around local release markers without raw Git execution. | Tag names, annotated tag messages, signatures, and object targets can leak release/customer/project intent. | Selected for a count/metadata-only proposal with raw names and messages suppressed. |

`git.show.tag_metadata` is the lowest-risk useful continuation of the existing bounded Git metadata
family. It can be implemented as local-only, fixed-argv Git inspection with no shell, no remotes, no
checkout, no mutation, no diffs, no file contents, no raw tag names, and no annotated tag messages.
