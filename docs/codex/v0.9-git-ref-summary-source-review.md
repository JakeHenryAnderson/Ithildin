## Review Questions

Internal xhigh review has already decided that `git.show.ref_summary` can continue for the
local-preview runtime boundary. Optional external review, if requested later, should focus on:

- structured selector validation and denial of unsupported selector fields;
- fixed Git argv behavior and no shell/checkout/remote-fetch/mutation behavior;
- bounded metadata output, absence of raw ref names, absence of stable ref-name hashes, and
  response-local `ref_id` semantics;
- parent-repository workspace-scope denial in preview and runtime;
- casefold-conflicting ref behavior, non-commit ref handling, and malformed Git metadata failures;
- policy preview/runtime parity for the syntactic `git_refs` resource;
- MCP exposure and role visibility through the existing governed pipeline;
- audit metadata staying limited to selector/count/truncation/output-policy evidence;
- no-new-powers and tool-surface gates staying limited to this bounded read-only addition.

## Internal xhigh Review Result

Internal xhigh source review found no blocking findings. Three low-severity hardening notes were
handled during implementation:

- safe execution audit metadata now records selector kind, counts, truncation, and output-policy
  booleans without raw ref names;
- `make no-new-powers-guardrail` now mirrors nested schema checks for the ref-summary selector;
- docs now clarify that response-local `ref_id` values are display handles, not privacy-preserving
  randomness or cross-response non-correlation guarantees.

With those notes addressed and focused/full gates passing, internal xhigh review is sufficient to
continue local-preview development for this bounded read-only lane. This is not public/security-
product approval and does not approve broader capability implementation.

## Closure Boundary

Even if this source-review lane closes, it does not approve:

- arbitrary Git execution;
- caller-supplied Git argv, format strings, pathspecs, refspecs, checkout, remotes, or credentials;
- raw ref names, stable ref-name hashes, raw diffs, file contents, blob contents, symlink targets, or
  broad filesystem access;
- write behavior, approval changes, production identity, hosted telemetry, runtime Postgres, remote
  MCP, public/security-product positioning, or future governed tool powers.

`git.show.ref_summary` remains local-preview read-only metadata mediation, not a Git sandbox.
